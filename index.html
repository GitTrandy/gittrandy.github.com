深入RSA

发表于 2015-11-22

RSA是典型的非对称加密算法，也是目前世界上最安全的加密算法。

其安全性的保证来自于目前的数学难题：大数的质因数分解的困难性。

数学上还没有找到一种好的方式快速的将一个大数分解成两个质数相乘的形式。

而在计算上，由于目前的机器性能问题，目前能被暴力验算分解的RSA密钥长度为663个二进制位数（参见），所以基本上我们认为RSA使用1024的密钥长度是安全的，极其需要安全性的可以使用长度为2048的密钥进行RSA加密。

当深入的去探究RSA的原理的时候，需要使用到一些数学知识，可以参看这里。

RSA加密流程：

  1. 我们找两个不相等的质数: p,q

  2. 计算p和q的乘积，记做n

  3. 计算n的欧拉函数ø(n) = (p-1)(q-1)

  4. 随机选择一个整数e使得e满足： 1<e<ø(n),GCD(e,ø(n)) = 1

  5. 计算e对于ø(n)的模逆元d: ed = 1 (mod ø(n))

  6. 将n和e封装成公钥，将n和d封装成私钥

  7. 使用公钥加密x(x为整数且xe = y(mod n),求出y

  8. 将加密后的结果y发送给对方，对方使用私钥解密。由数学上可以证明：yd = x (mod n)，所以计算可得x

    至此整个加密解密也就完成了。

由上面的简要流程我们可以看到实际上的加密解密都是数学运算,字符串可以取ASCII值或者Unicode值。

实践：

(1)首先我们随意找两个不相等的质数：13和17。这里我选择了两个比较小的质数，这是为了减少后面的计算量。通常来说我们真正使用RSA加密的时候会选择两个比较大的质数，这两个质数的乘积越大，则越难以破解。

(2)这样我们就得到: n = 13x17 = 221

(3)ø(n) = (13-1)*(17-1) = 192

这里ø表示的是欧拉函数，ø(n)就是求n的欧拉函数，

(4)由于需要找一个整数e满足GCD(e,ø(n)) = 1,(GCD表示求两个数的最大公约数,两个数的最大公约数为1表示两个数互质）

所以最简单的方法就是找个质数，那我们令 e = 23，这里也是简单举例，真正加密则会使用比较大的质数。

安全性

现在我们已经成功的简单的加密解密了一个数字，我们来考虑这样一个问题，为什么说RSA非常可靠的加密算法？

我们知道通常公钥是可以被拿到的，也就是n和e是被知道的。

那么我们只需要再知道d就可以破解私钥了。

我们有公式

ed = 1(mod ø(n))

知道ø（n）就知道d

而： n = p * q（p、q互质）

ø(n) = ø（p - 1） * ø(q - 1)

如果你能把n分解成两个质数p、q的乘积，那么就能求出ø(n)

但是目前来说大数分解是悬而未决的问题，当n的大小达到21024这种级别的时候,根本没办法分解(参见这里)

因此，我们认为RSA是安全可靠的，所以现在的RSA使用的加密密钥长度通常为1024，非常强调安全则使用2048长度的。

RSA相关数学知识

发表于 2015-11-22

这里只是简单简单介绍深入RSA需要知道的相关数学知识点，对于部分的知识点的介绍只挑出了部分，并不是全面，具体请参看链接。

质数(详细参见这里)：

质数，也叫素数。如果一个自然数有且只有两个正约数（1和此数本身），则称之为素数。

最大公约数(详细参见这里):

最大公约数，也叫最大公因数，简写为GCD，指几个整数公有约数中最大的一个。

欧拉函数(详细参见这里):

欧拉函数ø(n)是小于或等于n的正整数中与n互质的数的数目。
若m是质数，ø(m) = n - 1
若m,n互质，ø(mn) = ø(m) * ø(n)

模逆元(详细参见这里):

若有正整数a、b均小于n
则满足 ab = 1(mod n) 的b称为a对n的模逆元
a对n的模逆元存在的充要条件是a、n互质

扩展欧几里德算法(详细参见这里，代码实现见这里):

定理： 给予二整数a、b，必存在有整数x、y使得ax + by = gcd(a,b)
拓展欧几里德算法可对上面定理进行求解，亦可用于求模逆元

而扩展欧几里德算法则表述如下：

设 a,b 均为整数

ax + by = GCD(a,b) 总有整数解。

具体数学证明过程我们略过不谈，如果感兴趣可以参考这里。

我们来看这个等式，当a、b不互质的时候，也就是GCD(a,b) != 1。 因为GCD(a,b)是a，b的最大公约数，所以等式左右两边总可以约去公约数，最后会化简成 ax + by = 1的形式，其中a,b互质。这个问题也就变成了当a、b互质的时候，ax + by = 1总有整数解。

现在想办法对ax + by = 1求解。

上面欧几里德算法可知a、b互质则b、a%b互质，所以有

ax + by = 1

也有 bx1 + a%by1 = 1

那么可以得到：

ax + by = 1 = bx1 + a%by1 = bx1 + (a - (a/b)*b)y1

ax + by = ay1 + b(x1 - a/b*y1)

上面这个式子是恒等的，所以也就是 x = y1, y = x1 - a/b*y1。

这相当于确定了 x、y和x1、y1之间的关系。

那x1、y1又表示什么意思呢？

前面说过，x1、y1表示的是GCD(b,a%b) = 1的解。

那么整个推导串联起来就是：GCD(a,b) = 1的解x、y和GCD(b,a%b) = 1的解x1、y1有关，关系式为

x = y1, y = x1 - a/b*y1。

这条推论又可以递推下去：GCD(b,a%b) = 1的解x1、y1和GCD(a%b,b%(a%b)) = 1的解x2、y2有关，关系式为

x1 = y2, y1 = x2 - a/b*y2。

….

可以依次递推下去，那么最后一次递推应该是yn = 0,此时做边界判断。

由 GCD(k,0) = k,此时 kx + 0*y = k, 解为 x = 1 , y = 0。

我们可以得知上面推导步骤的最后边界条件应该是当yn = 0的时候，返回解 xn = 1，yn = 0，再由此逆推出x和y的解。
